A few things to note about form setup:

  • The 'action' is the URL the form information is sent to.
  • The 'name' is how the inforamtion is accesssed by the javascript.
  • Forms are processed by a server using PHP and the likes. Javascript can also process it before it's sent.

“var form = document.forms[0];” = “form = document.getElementsByTagname('form')[0]”. Either can be used to call on the form in Javascript. The number at the end can be replaced by ['the name of the form'].

Methods and events used with forms:

  • form.submit() submits the form (to the server?). This can be assigned to a button.
  • form.reset() resets the form back to its original settings. This can be assigned to a button. Rest buttons aren't considered best practice, sine they're easy to accidentally click.
  • form.action = 'URL' can change the URL the form is submitting to.
  • “var form = document.forms.search;
    form.addEventListener ("submit", search, false);
    function search() {
    alert("Submitted");
    }”

    This code listens for the submit action and will send the form to hte Javascript before it's sent on to the server to be processed.
  • event.preventDefault(); prevents a default action from being performed, such as submitting the form.
