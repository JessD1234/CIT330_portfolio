Reading about debugging this week. Here are a few things that stood out to me that I want to remember:

  • There is a strict mode for Javascript! I need to learn more about this and if it'll be useful. One of the hardest part about javascript is the fact that it's so easy to make an error and not know it. It makes it very frustrating to try and use the language. To access strict mode, just add this line to the beginning of a Javascript file: 'use strict';
  • You can use alerts in the code as manual break points, since these alerts pause the code and the code won't keep running into the ok button is clicked. It seems this method would give you a more in depth look at your code when combined with debugging in the browser.
  • Adding 'debugger' to code works similarly.
  • Remember, any in-code debugging tools must be removed before the final product is implemented, or it'll cause issues for the user!
  • Functions that return values if the answer is true can be used to test if a function is working right. For example, to test a function that returns the square root, you could run the result of that function through another function that will return true if the result is equal to the correct answer. This allows you to insert a number into your first function, run it, check it against the 'answer key' function, and then see if it ran correctly.
  • Test driven coding process:
    1. Write tests (that initially fail)
    2. Write code to pass the tests
    3. Refactor the code
    4. Test refactored code
    5. Write more tests for new features
