Here's some notes about form verification:

  • Form verification happens before the form is sent to the server. It can help speed up the submission process by checking the form as the user goes, instead of making them wait for an error to come back.
  • Form verification can be used as a security measure, but shouldn't be relied on. It only checks that the data looks clean. Security measures should be implemented on the server side to double check.
  • The best use of form validation is to make sure the data is in the correct format, so when it's submitted to the server it can be stored correctly. This prevents errors in future uses of the data.
  • Form verification can be built in to hte HTML or it can be written in Javascript. Here are some examples:
    • required - marks the field required. It must be filled out before the form can be submitted.
    • minlength and maxlength - sets the length required for that field. Specific for text strings
    • min and max - same as above, but for other data types.
    • pattern - requires the data to conform to a certain preset pattern. (How to set these?)
    • Javascript has similar elements as the above listed.
