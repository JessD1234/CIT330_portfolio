Here's some notes about window objects:

    The window object is equivalent to the global object when working in a browser environment.
  • window.navigator.userAgent will return information about the browser and machine the user is accessing the page on.
  • window.location access information about the URL.
  • window.location.href gets the URL of the page. New values can be assigned to this, and when the page reloads, it'll redirect to whatever is stored there.
  • window.location.protocol returns the protocol being used.
  • window.location.host will show where the website is being hosted.
  • window.location.reload() forces a reload from the server instead of using a cached version. (Possibly useful for testing so I don't get caught on hard refreshes when the dev tools aren't open?)
  • window.history can show a list of pages visited during the current browsing session. (sketchy! Haha). Add .length and it'll count how many were visited before arriving at the current page.
  • window.screen gives you insights into the screen size, length, width, etc.
  • document.cookie gives you some options with cookies attatched to the page.

Some notes about HTML APIs:

  • data- can be used to add basically custom attirbutes to an object. They're just for using in javascript. This week's group activity used these!
  • Web storage is basically an upgraded cookie.
  • The geolocation API gets a user's location, and other combinations with like navigator and such can get different information.
  • Web workers allows more than one javascript function to run at a time, allowing for larger programs to be pushed to the background so the page can load.
